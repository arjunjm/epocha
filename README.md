# Epocha

An AI-powered historical timeline generator. Enter any topic and get a rich, structured timeline with events, key figures, significance, and quiz questions — all generated by a large language model and cached for instant repeat access.

**Live:** https://app-timelineapp-dev.azurewebsites.net

## LLM Provider

The app abstracts the LLM provider via `server/src/llm.ts`. Switch providers by setting a Key Vault secret:

```
llm-provider = azure-openai   # default
llm-provider = anthropic
```

Azure OpenAI requires three additional secrets: `azure-openai-endpoint`, `azure-openai-key`, `azure-openai-deployment`.

## Caching Strategy

Timeline responses are cached in Redis with a 7-day TTL using the key:

```
timeline:<normalised-topic>:<startYear>:<endYear>
```

Example: `timeline:the-roman-empire:27 BCE:476 CE`

**Cache layers:**
- `GET /api/timeline/browse` — public endpoint, serves cache only, no auth required
- `POST /api/timeline` — authenticated endpoint, checks cache first; only calls LLM on miss; rate limit only increments on actual LLM calls
- Nightly Azure Function — pre-generates and caches the full topic catalog overnight
- `epocha:trending-topics` sorted set — tracks recently generated topics for the sidebar Trending section
- `epocha:popular-topics` sorted set — tracks search frequency per topic for demand-driven pre-caching

## Azure Function Triggers

| Trigger | Method | Description |
|---------|--------|-------------|
| `pregenerateTimelines` | Timer (2AM UTC) | Nightly full queue: trending events → popular searches → AI-suggested → sidebar |
| `pregenerateManual` | POST HTTP | Manual trigger for the full dynamic queue |
| `generateTrendingEvents` | POST HTTP | Ask LLM for today's 10 trending current events, generate and cache timelines |

The nightly queue phases (in order):
1. **Trending current events** — LLM asked for 10 significant contemporary events with today's date
2. **Popular searches** — top 30 from `epocha:popular-topics` + their `relatedTopics`
3. **AI-suggested** — 15 historically interesting topics suggested by the LLM
4. **Sidebar defaults** — all ~40 taxonomy topics kept fresh

## User Features

- **Timeline generation** — topic + optional date range → streamed JSON, events appear progressively
- **Quiz** — 12 MCQs generated per timeline, 5 served per session; XP rewards
- **Gamification** — 20 levels, XP system, 8 achievement badges, Steam-style showcase
- **Bookmarks** — save individual events across sessions (localStorage)
- **Collections** — save full timelines to named collections (Cosmos DB); reorder by drag & drop
- **Flashcard mode** — memory challenge: reveal events one by one, score yourself
- **Time Machine** — fullscreen immersive event slideshow
- **Key Figure Explorer** — click a figure to filter the timeline to their events
- **Insights panel** — analytics: events by century, top figures, themes, locations
- **Density heatmap** — visualise event distribution across eras
- **Related events** — each expanded event shows similar events from the same timeline
- **Reading progress** — marks events as read, shows progress bar
- **Personal notes** — per-event text annotations saved to localStorage
- **Export** — PDF (print), Markdown, self-contained HTML
- **Themes** — 5 visual themes (Midnight, Sepia, Neon, Ocean, Forest)
- **Light/dark mode** — toggle in nav bar, no login required
- **PWA** — installable, offline-resilient

## Local Development

**Prerequisites:** Node.js 18+, an Anthropic or Azure OpenAI API key

```bash
# Install all dependencies
npm install
cd server && npm install
cd ../client && npm install
cd ../functions && npm install

# Server env (server/.env)
ANTHROPIC_API_KEY=sk-ant-...   # or use Azure OpenAI (see below)

# Run dev servers (client on :5173, server on :3001)
npm run dev
```

Open http://localhost:5173. The server falls back to in-memory cache when no `REDIS_URL` is set, and local JSON files when no `COSMOS_ENDPOINT` is set.

**Switching to Azure OpenAI locally:**
```bash
# server/.env
LLM_PROVIDER=azure-openai
AZURE_OPENAI_ENDPOINT=https://your-resource.openai.azure.com/
AZURE_OPENAI_KEY=your-key
AZURE_OPENAI_DEPLOYMENT=gpt-4o
```

## Deployment

Deployments are triggered automatically on push to `master` via GitHub Actions (`.github/workflows/deploy.yml`):

1. **Test** — runs Vitest for server and client
2. **Build** — compiles TypeScript, builds Vite client
3. **Deploy App Service** — uploads to Azure App Service (Oryx builds `node_modules`)
4. **Deploy Functions** — zips `dist/` + `node_modules/` → uploads to blob storage → sets `WEBSITE_RUN_FROM_PACKAGE`

**Required GitHub secrets:** `AZURE_WEBAPP_PUBLISH_PROFILE`, `AZURE_CLIENT_ID`, `AZURE_CLIENT_SECRET`, `AZURE_TENANT_ID`, `AZURE_SUBSCRIPTION_ID`, `AZURE_STORAGE_KEY`

**Key Vault secrets (loaded at startup):**
```
anthropic-api-key         llm-provider
azure-openai-endpoint     azure-openai-key        azure-openai-deployment
google-client-id          google-client-secret    google-callback-url
jwt-secret                cosmos-endpoint         cosmos-key
redis-url
```
